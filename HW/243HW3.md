a) Utility Class

System.Net.WebUtility
จาก .NET API (Utility class สำหรับงานทั่วไป)

string encoded = WebUtility.UrlEncode("a+ b");
string decoded = WebUtility.UrlDecode(encoded);
ไม่สามารถสร้างอินสแตนซ์ เพราะเป็น static class

เพราะเหตุใดถึงเรียกว่า Utility Class?

เป็นคลาสที่เป็น static และมีแต่ method ช่วยเหลือ (เช่น การเข้ารหัส URL, การถอดรหัส HTML) ไม่ได้เก็บข้อมูลสถานะภายในตัวเอง
มักใช้เป็น helper ในงานทั่วไปโดยไม่ต้องสร้างอินสแตนซ์ของคลาส

b) Mutable Class

System.Collections.Generic.HashSet<T>
เป็นคลาสคอลเลกชันที่คุณสามารถเพิ่ม/ลบสมาชิกได้

var set = new HashSet<int>();
set.Add(1);       
set.Remove(1);    

เหตุใดจึงเป็น Mutable Class?

สามารถ แก้ไขสถานะภายในของออบเจ็กต์หลังสร้างแล้ว ได้โดยตรง (Add, Remove, Clear)
สภาพออบเจ็กต์เปลี่ยนได้ตลอดเวลา

c) Immutable Class

System.Collections.Immutable.ImmutableList<T>
คลาสนี้สร้าง collection ที่ ไม่เปลี่ยนค่า หลังสร้างออบเจ็กต์แล้ว

var list = ImmutableList.Create("a", "b", "c");
var newList = list.Add("d"); 
// list ยังคงเดิม, newList เป็น object ใหม่

เหตุใดจึงเป็น Immutable Class?

หลังสร้างแล้ว ออบเจ็กต์ไม่สามารถเปลี่ยนแปลงสถานะภายในได้โดยตรง
ทุก method ที่ดูเหมือน mutating (เช่น Add) จะคืน object ใหม่ แทนที่จะเปลี่ยน object เดิม
