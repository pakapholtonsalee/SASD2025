เล่าว่าใน20ปีที่ผ่านมาเขาพยายามพัฒนา code ให้เข้าใจง่ายจัดการง่าย โดยปัญหาที่เขาเอามาพูดคือเรา
จะปรับปรุงโค้ดให้ดีขึ้นได้ยังไง โดยเขาเสนอการ Refactoring/Tidy มาช่วยปรับปรุงโค้ดโดย ไม่ทำให้
โปรแกรมดั้งเดิมเปลี่ยนไปและจะทให้โค้ดอ่านได้ง่ายมากยิ่งขึ้นแล้วจะช่วยให้จัดการได้สะดวกขึ้นด้วย การ 
tidy code ควรทำอย่างมีเหตุผล และไม่ใช่ทำตามสูตรตายตัวเพื่อให้โค้ดง่ายต่อการเปลี่ยนแปลงในระยะ
ยาวได้สะดวกหากต้องการจะปรับปรุง
